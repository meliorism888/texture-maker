# texture/maker — procedural pattern studio

A browser-based studio for generating seamless, tileable procedural textures.
Twelve render engines, live parameter controls, color palettes, post-FX,
a saved library, and PNG / CSS / data-URI / JSON export.

## Quick start (local)

```bash
npm install
npm run build   # compiles + bundles to dist/
npm run serve   # serves dist/ at http://localhost:3000
```

For live development:
```bash
npm run watch   # rebuilds on every save
```
Then open `dist/index.html` in a browser (or use `npm run serve`).

## Deploy

### Netlify (recommended — one click)
1. Push this repo to GitHub
2. Connect repo to Netlify at netlify.com
3. Build command and publish dir are already set in `netlify.toml`
4. Done — auto-deploys on every push

### Other static hosts (Vercel, GitHub Pages, Cloudflare Pages, S3)
Run `npm run build`, upload the `dist/` folder. No server required.

## Performance

The source `index.html` used in-browser Babel transpilation (slow).
The build pipeline uses esbuild to precompile JSX and bundle React:

| Before (CDN + runtime Babel) | After (esbuild bundle) |
|-----------------------------|------------------------|
| ~6 MB transferred            | ~220 KB transferred    |
| JSX transpiled on every load | Pre-compiled at build  |
| React dev warnings in console| Production React build |

## Key features

- **Seamless tiling** — toroidal edge-wrapping + tileable noise so every
  export repeats with no visible seam. Verify with the tile preview (T).
- **12 engines** — Dots, Hatching, Grid, Noise, Voronoi, Scatter, Waves,
  Truchet, Flow, Hex, Bricks, Rings.
- **Deterministic seeds** — same seed + params always reproduce the texture.
- **Share URLs** — full state encodes into the URL hash.

## Shortcuts

`Space` randomize · `E` export PNG · `T` tile preview · `I` invert ·
`⌘S` save to library · `⌘Z` / `⇧⌘Z` undo / redo · `?` help
